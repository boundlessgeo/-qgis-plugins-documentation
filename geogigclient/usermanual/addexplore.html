4. Viewing Data and Exploring History¶


GeoGig Quick Start - Looking At Data

In this section, we will be looking at adding GeoGig layers to QGIS and viewing the history of a repository.

4.1. GeoGig Server VM sample Data¶

The GeoGig Server VM comes with several layers of Open Street Map Data (OSM) from Geofabrik.de.

The dataset consists of these layers for the state of Missouri:

Layer Description
Buildings Building footprints
Roads Roads and highways
water_area polygons representing lakes and large rivers
water_line lines representing rivers
landuse polygons representing primary use of land by humans

It also contains snapshots of the OSM dataset at 10 points in time:

Snapshot Date
January 1, 2014
January 1, 2015
January 1, 2016
January 1, 2017
January 1, 2018
July 1, 2018
August 1, 2018
August 16, 2018
September 1, 2018

NOTE: these are the state of the actual OSM dataset at these points in time. The OSM dataset was started in 2004 and is continuously updated.

4.2. Add a GeoGig Layer to QGIS¶

You can add a GeoGig layer to your QGIS project very easily through either the Navigator or the Constellation Viewer.

You can, also, add an older revision of the dataset using the History Viewer (see below).

4.2.1. In the Navigator¶

  1. Use the Navigator’s tree to open the repository that contains the layer you want.
  2. Look in the Layers folder.
  3. Either
    • Click either Add to QGIS project (live link) or Add to QGIS project (geopackage) in the info section (below the tree)
    • Right-click on the layer you want to add and select either Add to QGIS project (live link) or Add to QGIS project (geopackage).

See GeoPKG Layers for more information about the difference between GeoPKG and Live layers. See below for how to add historical versions of the dataset.

4.2.2. In the Constellation Viewer¶

  1. Use the Navigator to open the Constellation Viewer (right-click on a repository and choose Show Repo Constellation).
  2. In the the Constellation Viewer, find the repository you want the layer to come from.
  3. In the Repository’s Layer section, find the layer you are interested in.
  4. Either
    • Click on the Add as Live Link icon (document connected to network).
    • Click on the Download GeoPKG icon (cloud with a arrow pointing down).

See GeoPKG Layers for more information about the difference between GeoPKG and Live layers. See below for how to add historical versions of the dataset.

4.3. Viewing History¶


GeoGig Quick Start - Viewing History

The History Viewer lets you see all the changes made to a repository or layer. It also allows you to “travel in time” and see what the dataset looked like in the past.

You can open the History Viewer either from the Navigator window or the Constellation Viewer. The history viewer can be either opened at the repository level (showing all changes for all layers) or at an individual layer level (only showing changes that affect a single layer).

4.3.1. The History Viewer¶

The History Viewer

The History Viewer has two main parts;

  • Commit Viewer (at the top) - this shows all the commits (changesets) made to the repository/layer
  • Commit Details (at the bottom) - there are three tools for looking at a commit’s details (Commit Summary, Map, Attributes)

4.3.2. The History (Commit) Graph/List¶

The Commit Graph/List is at the top of the History Viewer. The main portion has a list of each commit, as well as a graph showing the connection between commits.

Each commit has the following information:

Graph graphical history of commits (i.e. Pull Requests)
Description the description the user entered when making the commit
Author who made the commit
Date when the commit was made
CommitID GeoGig unique ID used to identify this commit

Typically, the branching/merging of the graph indicates what the state of the datasets were when an independent workspace (repository) was created to hold changes for a workflow. A merging commit in the graph indicates that a Pull Request was made and accepted. See the workflow and synchronization sections and the Introduction to GeoGig, Quick Start, and Synchronization videos for details of interpreting commit history graphs.

There are also two tools, at the very top;

  • Search Box - this is to help find commits. Type in some text here to filter the commit list
  • Show Simplified History - this shows the history as a simple list of changes (all the workflow - merge - commits are not shown)

4.3.3. Exploring a Commit¶

The bottom section of the History window shows details about the commit selected in the top (Graph/List) section.

There are three ways to look at this commit - Commit Summary tab, Map tab, and the Attributes tab.

The Map and Attributes tab are only available if the commit modified less than 1500 features (for large commits, use the tools available in the Commit Summary tab).

4.3.3.1. Summary Tab¶

The Commit Summary tab

The Summary tab gives more information about a single commit selected in the Graph/List. For each layer, it gives you the number of features that that this commit added, deleted, and modified. It also allows you to;

  • Add Live - add a live layer to QGIS that will show the layer as it was after this commit was made (time-travel)
  • Add GeoPKG - download a GeoPKG layer to QGIS that will show the layer as it was after this commit was made (time-travel)
  • Export Diff - Add a layer to QGIS that details all the changes to the layer introduced by this commit (see below)

There is also an Export commit’s DIFF for all layer button that will add a diff layer for each layer modified in the commit (see below).

4.3.3.2. Attribute Tab¶

The Attribute Tab

The Attribute tab shows the details of all features changed in a single commit. It has two parts - the feature tree (left) and change details (right).

Select a feature in the feature tree and the change details section will show the feature before the commit (Old Value) and after the commit (New Value). This allows you to dive deep into the changes introduced during a commit.

NOTE: this tab is only available if less than 1500 features were modified by the commit.

4.3.3.3. Map Tab¶

The map view shows an overview map of all the features modified in this commit.

NOTE: this tab is only available if less than 1500 features were modified by the commit.

4.3.4. Exporting a Commit Diff¶

If a commit has a lot of changes, it is beneficial to export the Diff (“Difference”). This can be done in two ways;

  • Single Commit Diff - press the Export Diff or Export commit’s DIFF for all layer button. This will show the differences made by a single commit.
  • Select two commits in the Graph/List, right-click and select Add DIFF layers to QGIS for these commits. This will show the differences made between any two commits.

Selecting any two commits then right-click to see the differences between them

The resulting diff layer

The diff layer will show all the changes in three colors:

  • Adds - in green
  • Deletes - in red
  • Modification - in two shades of yellow (both the before-change and after-change feature will be shown)

Attribute table for the diff layer

View the diff layer’s attribute table for more details. The attribute table will show, for each changed feature, what kind of change was done (Add, Delete, or Modify) - in the “GeoGig.ChangeType” column. It also shows all the attributes for the feature. For a modification, there are two entry rows in the attribute table - the feature as it was before the change and the feature as it was after the change.

NOTE: exporting a large diff can take some time (several minutes).
